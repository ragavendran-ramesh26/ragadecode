New UPI Rules from August 1, 2025: What Every User Must Know

WhatsApp Twitter Facebook

Published on: 17 Jul 2025

Who Announced the New UPI Rules?

The National Payments Corporation of India (NPCI), which operates the UPI (Unified Payments Interface) platform, announced a series of usage restrictions and API control measures. These are applicable to all banks, payment service providers (PSPs), and UPI users.

NPCI is responsible for ensuring the stability, scalability, and security of India’s digital payment infrastructure.

What Are the New UPI Rules?

The following key changes will come into effect from August 1, 2025:

  • Balance Check Limit: Max 50 per app per day
  • Linked Account Fetch Limit: Max 25 times per app per day
  • Transaction Status Check: Only 3 times per transaction, with a 90-second gap
  • AutoPay Scheduling: Will only run during non-peak windows (before 10 AM, 1–5 PM, after 9:30 PM)
  • API Response Time Benchmarks: 15 sec (payments), 10 sec (status checks)

When Will These Changes Take Effect?

All UPI ecosystem participants must implement these rules by July 31, 2025, and the enforcement begins August 1, 2025. NPCI has issued a compliance deadline, and non-compliance could result in penalties, service throttling, or temporary suspension.

Where Will These Rules Apply?

These changes apply nationwide across India, covering:

  • All UPI apps (Google Pay, PhonePe, Paytm, BHIM, Amazon Pay, etc.)
  • All banks integrated with the UPI network
  • Every individual and merchant using UPI

International UPI partnerships (like UPI-Singapore and UPI-UAE) are not immediately affected by these limits unless explicitly stated by NPCI.

Why Were the New Rules Introduced?

The rapid growth of UPI—handling over 12 billion transactions monthly—has resulted in:

  • Server overloads
  • Payment delays
  • Frequent system downtimes
  • High volume of API hits from PSPs

To ensure system stability and prepare for further scale (including credit card UPI, international payments, and voice-enabled UPI), NPCI is optimizing backend infrastructure by enforcing controlled access to heavy-load APIs.

Conclusion

The new UPI rules effective August 1, 2025, mark a critical step in India's digital payment evolution. While they introduce limitations, they are designed to improve performance, avoid outages, and ensure a seamless experience for billions of transactions.

Quick Info

Are PSPs and banks affected by the new UPI limits?
Yes, all payment service providers (PSPs) and banks on the UPI network must comply with the new daily API and usage limits.
Who announced the new UPI rules for 2025?
The new UPI rules were announced by the National Payments Corporation of India (NPCI), which operates and regulates the UPI platform in India.
When will the new UPI rules take effect?
The new rules will be enforced starting August 1, 2025. All UPI participants must implement changes by July 31, 2025.
What is the new limit for balance checks on UPI apps?
Users can now check their bank balance a maximum of 50 times per day per UPI app.
How many times can users fetch linked account information per day?
Users can fetch linked account details up to 25 times per UPI app daily.
What are the new rules for checking transaction status on UPI?
A status check can only be done 3 times per transaction, with a mandatory 90-second gap between attempts.
What changes are being made to UPI AutoPay scheduling?
UPI AutoPay schedules will only be executed during non-peak hours — before 10 AM, between 1–5 PM, and after 9:30 PM.
What is the new benchmark for UPI API response times?
The response time benchmarks are 15 seconds for payments and 10 seconds for transaction status checks.
Why is NPCI enforcing these new UPI rules?
The changes aim to address issues like server overload, transaction delays, and API abuse, while preparing the system for growth in international UPI, credit card integration, and voice payments.
How many UPI transactions are currently processed monthly?
UPI handles over 12 billion transactions every month, leading to considerable backend strain.
What happens if a UPI participant fails to comply with the new rules?
Non-compliance can lead to penalties, API throttling, or even temporary service suspension by NPCI.
Which platforms will these UPI changes affect?
The changes apply to all UPI platforms, including Google Pay, PhonePe, Paytm, BHIM, Amazon Pay, and more.
Are international UPI integrations affected by these changes?
No, UPI partnerships like UPI-Singapore and UPI-UAE are not currently affected unless NPCI issues specific guidance.
Will these changes affect merchants who use UPI?
Yes, merchants using UPI are also subject to these API and usage limitations to ensure consistent performance across the board.
What is the benefit of limiting balance checks to 50 per day?
It helps reduce unnecessary load on banking APIs, promoting efficient use and improving system speed and reliability.

In-Depth Answers

How will users be impacted by the 3-attempt limit on transaction status checks?
Users must wait at least 90 seconds between each status check and can only try 3 times per transaction, which may prevent spamming the system during delays.
What are the peak and non-peak times for UPI AutoPay operations?
Non-peak hours are defined as before 10 AM, between 1–5 PM, and after 9:30 PM. AutoPay transactions will only process during these times.
What kind of infrastructure improvements are these rules targeting?
The rules aim to reduce backend strain, prevent outages, and enhance system stability as UPI scales toward more advanced use cases.
Will the new rules speed up UPI performance?
Yes, by regulating heavy API usage and transaction loads, the rules are expected to result in faster response times and fewer downtimes.
How do the new UPI rules promote responsible usage?
By capping API calls and status checks, the system discourages excessive, automated queries, leading to more efficient and fair use of UPI resources.
What’s the main goal of NPCI’s 2025 UPI update?
The update is aimed at ensuring scalability, reducing system stress, and maintaining a seamless experience as UPI adoption continues to grow across India and abroad.
Do the rules change how often users can access their UPI-linked accounts?
Yes, account-fetch APIs are now limited to 25 uses per day per app, reducing redundant calls to banking servers.
Will users still be able to make unlimited payments through UPI?
There is no mention of a cap on the number of payments, but associated API calls for status checks and balance views are limited.
Is the 90-second gap between transaction status checks mandatory?
Yes, a 90-second delay is required between successive status checks to prevent server overload and redundant queries.
How do these UPI changes affect automated scripts or bots?
The new API usage caps and enforced delays directly curb the effectiveness of bots or automated scripts that generate high volumes of API requests.
Are these rules permanent or subject to future revision?
While these rules are set for August 2025, NPCI may revise them based on infrastructure readiness, user feedback, and future UPI capabilities.
